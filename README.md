# 灾难救援翻译平台

## 项目概述

本项目是一个专为跨文化灾难救援任务设计的翻译平台，旨在帮助中国救援队与缅甸灾民进行有效的沟通。平台集成了先进的大语言模型，提供实时翻译服务，并特别关注文化敏感性和情绪感知，同时支持离线模式以应对电缆损坏等网络问题。

## 核心功能

### 1. 实时翻译
- 支持中文、缅甸语、英语之间的相互翻译
- 集成DeepSeek大语言模型，提供高质量翻译
- 响应迅速，适合紧急救援场景

### 2. 文化敏感性分析
- 针对中缅文化差异进行优化
- 识别可能引起情绪波动的敏感词汇
- 提供文化注释和沟通建议
- 避免高低语境差异带来的误解

### 3. 离线支持
- 自动缓存翻译结果
- 网络中断时自动切换到离线模式
- 支持缓存管理和查询
- 减少对网络依赖，应对电缆损坏情况

### 4. 情绪感知
- 识别灾民情绪状态
- 提供情绪友好的翻译结果
- 避免使用可能加重灾民心理负担的词汇

## 技术栈

### 后端
- **Node.js** + **Express.js** - 服务器框架
- **Axios** - HTTP客户端，用于调用DeepSeek API
- **DeepSeek API** - 提供翻译和文化分析服务
- **SQLite** - 本地缓存存储
- **Node-localstorage** - 离线数据存储

### 前端
- **React 18** - UI框架
- **Material UI (MUI)** - 组件库
- **Vite** - 构建工具
- **Axios** - HTTP客户端
- **React Toastify** - 通知组件

## 安装和运行

### 1. 安装依赖

```bash
# 安装所有依赖
npm run install:all
```

### 2. 配置DeepSeek API

在`backend/server.js`文件中配置你的DeepSeek API密钥：

```javascript
const DEEPSEEK_API_KEY = process.env.DEEPSEEK_API_KEY || 'your-deepseek-api-key';
```

### 3. 启动开发服务器

```bash
# 同时启动前端和后端
npm run dev

# 或者分别启动
npm run dev:backend  # 后端运行在3001端口
npm run dev:frontend # 前端运行在3000端口
```

### 4. 访问应用

打开浏览器访问：http://localhost:3000

## 使用说明

1. **选择语言**：在源语言和目标语言下拉菜单中选择需要翻译的语言对
2. **输入文本**：在左侧文本框中输入要翻译的内容
3. **开始翻译**：点击"开始翻译"按钮
4. **查看结果**：在右侧查看翻译结果和文化注释
5. **注意文化提醒**：如果有文化敏感性问题，会显示警告信息
6. **离线模式**：当网络断开时，系统会自动切换到离线模式

## 文化敏感性配置

平台内置了中缅文化敏感性配置，包括：

### 缅甸文化
- 高语境文化
- 敏感词汇：地震、损失、家人、家园
- 尊敬用语：先生、女士、朋友
- 避免短语：没问题、随便、不用担心

### 中国文化
- 高语境文化
- 敏感词汇：救援、安全、希望、重建
- 尊敬用语：同志、师傅、家人
- 避免短语：没办法、不可能、等一下

## 网络适应策略

1. **自动检测网络状态**
2. **翻译结果缓存**
3. **离线模式切换**
4. **减少网络请求**
5. **优化数据传输**

## 救援沟通小贴士

1. 使用简单、清晰的语言
2. 保持语气温暖、尊重
3. 避免使用可能引起情绪波动的词汇
4. 注意肢体语言和文化差异
5. 倾听灾民需求，给予情感支持
